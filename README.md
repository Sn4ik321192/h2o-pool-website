# H2O Pool Website

Современный адаптивный сайт меню для заведения у бассейна H2O в стиле Liquid Glass. В проекте есть главная страница, меню с данными из Supabase и админ-панель для добавления, редактирования и удаления категорий и товаров.

## Стек

- React + Vite
- Supabase PostgreSQL + Supabase Auth
- CSS Liquid Glass UI
- Lucide React icons

## Запуск локально

1. Установите зависимости:

```bash
npm install
```

2. Создайте файл `.env` на основе `.env.example`:

```bash
VITE_SUPABASE_URL=https://your-project-ref.supabase.co
VITE_SUPABASE_ANON_KEY=your-public-anon-key
```

3. Запустите проект:

```bash
npm run dev
```

4. Откройте адрес из терминала, обычно `http://localhost:5173`.

## Настройка Supabase

### 1. Регистрация

1. Откройте [supabase.com](https://supabase.com).
2. Нажмите **Start your project**.
3. Войдите через GitHub или зарегистрируйтесь по email.

### 2. Создание проекта

1. В Supabase Dashboard нажмите **New project**.
2. Выберите организацию.
3. Назовите проект, например `h2o-pool-website`.
4. Создайте надежный database password и сохраните его.
5. Выберите ближайший регион.
6. Нажмите **Create new project**.

### 3. Создание таблиц

1. Откройте проект в Supabase.
2. Перейдите в **SQL Editor**.
3. Нажмите **New query**.
4. Вставьте SQL из файла `supabase-schema.sql`.
5. Нажмите **Run**.

Основные таблицы:

- `categories`: `id`, `name`, `created_at`
- `products`: `id`, `name`, `description`, `price`, `image_url`, `category_id`, `created_at`

Минимальная структура из задания:

```sql
create table categories (
  id bigint generated by default as identity primary key,
  name text not null unique
);

create table products (
  id bigint generated by default as identity primary key,
  name text not null,
  description text,
  price numeric(10, 2) not null,
  image_url text,
  category_id bigint references categories(id)
);
```

### 4. Полный SQL с RLS и тестовыми данными

Используйте файл `supabase-schema.sql`. Он создает таблицы, включает Row Level Security, разрешает публичное чтение меню и разрешает управление товарами только авторизованным пользователям.

### 5. Получение API URL и ключа

1. В Supabase Dashboard откройте **Project Settings**.
2. Перейдите в **API**.
3. Скопируйте **Project URL**.
4. Скопируйте **anon public key**.
5. Вставьте их в `.env`:

```bash
VITE_SUPABASE_URL=ваш_project_url
VITE_SUPABASE_ANON_KEY=ваш_anon_public_key
```

### 6. Создание администратора

1. В Supabase Dashboard откройте **Authentication**.
2. Перейдите в **Users**.
3. Нажмите **Add user**.
4. Укажите email и пароль.
5. Этими данными войдите на вкладке **Админ** на сайте.

## Как пользоваться админ-панелью

1. Откройте скрытую ссылку админ-панели: `https://sn4ik321192.github.io/h2o-pool-website/?admin=1`.
2. Локально можно открыть: `http://localhost:5173/?admin=1`.
3. На компьютере также работает скрытый быстрый вход: зажмите `Alt` и нажмите на логотип **H2O**.
4. На телефоне удерживайте логотип **H2O** примерно 2 секунды.
3. Войдите через email и пароль пользователя Supabase Auth.
4. Добавьте категории.
5. Добавьте товары, указав название, описание, цену, URL фото и категорию.
6. Редактируйте или удаляйте товары через список ниже формы.

## GitHub Pages

В проект уже добавлен workflow `.github/workflows/deploy.yml`. После пуша в `main` GitHub Actions соберет сайт и опубликует его через GitHub Pages.

Перед деплоем добавьте Supabase ключи в GitHub:

1. Откройте репозиторий на GitHub.
2. Перейдите в **Settings** -> **Secrets and variables** -> **Actions**.
3. Создайте два repository secrets:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
4. Перейдите в **Settings** -> **Pages**.
5. В **Build and deployment** выберите **GitHub Actions**.

Локально production-сборку можно проверить так:

```bash
npm run build
```

Готовые файлы появятся в папке `dist`.
